# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 1.2.x   | ✅ Active           |
| 1.1.x   | ✅ Security fixes   |
| 1.0.x   | ⚠️ End of life      |
| < 1.0   | ❌ Unsupported      |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Email: **security@agentdyne9.com** (or **hello@agentdyne9.com** if security alias not yet active)

We acknowledge reports within **48 hours** and target resolution of critical issues within **14 days**.

## Scope

In scope: remote code execution via API, authentication bypass, injection
vulnerabilities (prompt, SQL, shell), sensitive data exposure, privilege
escalation via RBAC, denial-of-service via resource exhaustion.

Out of scope: social engineering, physical security, issues in third-party
dependencies (report those upstream).
